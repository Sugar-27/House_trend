# 中国房屋市场趋势分析项目：产品与技术方案

## 1. 项目目标

构建一个可通过网页访问的中国房屋市场趋势分析项目，首期聚焦上海市，后续保留扩展到其他城市的能力。项目需要同时适配 PC 端和移动端，提供现代化、精致、高级感的可视化体验。

核心分析对象包括：

- 房屋价格趋势
- 房屋成交量趋势
- 房屋购买量 / 销售量趋势
- 成交面积
- 成交总金额
- 房贷利率、首付比例等楼市金融指标

## 2. 数据来源

PRD 中指定的主要数据来源：

1. 网上房地产：https://www.fangdi.com.cn/
2. 上海银行间同业拆放利率：https://www.shibor.sh.cn/shibor/
3. 上海公积金官网：https://www.shzfgjj.cn/

首期以 Mock 数据打通页面和交互闭环，后续再接入真实采集、清洗、入库与聚合查询流程。

## 3. 是否需要本地存储数据

结论：需要本地存储数据。

原因：

1. 趋势分析依赖历史数据沉淀，仅实时请求外部网站无法支持长期趋势、同比、环比等分析。
2. 外部数据源未必提供稳定公开 API，且存在 CORS、访问频率、页面结构变化等风险。
3. 趋势报告、区域对比、月度汇总等适合后端预计算或聚合后提供。
4. 安全、限流、防刷、接口保护均需要后端能力。

因此，最终架构建议采用前后端分离：前端负责展示，后端负责采集、清洗、存储、聚合与安全控制。

## 4. 推荐产品形态

### 4.1 首页

首页作为综合看板，提供快速概览和功能入口。

建议包含：

- 城市筛选，首期默认上海
- 房屋类型筛选：一手房 / 二手房
- 时间范围筛选：近 30 天、近 90 天、近 1 年
- 核心指标卡片：成交均价、成交套数、成交面积、成交金额
- 趋势图：价格趋势、成交量趋势、成交面积趋势
- 区域成交排行
- 市场分析摘要
- 子页面入口：成交数据、趋势分析报告、楼市利率

### 4.2 成交数据页

成交数据页以日历为核心。

建议包含：

- 月份筛选
- 城市、区域、房屋类型筛选
- 日历视图
- 每日成交套数、成交面积、成交均价、成交金额
- 月度汇总栏
- 点击日期后的详情扩展能力

### 4.3 趋势分析报告页

报告页展示城市整体和区域维度的趋势。

建议包含：

- 城市整体趋势
- 区域趋势对比
- 房价趋势
- 成交量趋势
- 成交面积趋势
- 自动生成的规则型文字报告

### 4.4 楼市利率页

楼市利率页展示贷款和购房金融相关指标。

建议包含：

- 商业贷款利率
- 公积金贷款利率
- 首套房 / 二套房利率
- 首付比例
- SHIBOR / LPR 参考趋势
- 房贷成本说明，后续可扩展贷款计算器

## 5. 技术方案

### 5.1 总体架构

```text
用户浏览器
  ↓
Next.js 前端 / Vercel
  ↓
Next.js BFF 层 / API Proxy
  ↓
Go 后端服务
  ↓
MySQL 数据库
  ↓
定时采集任务
  ↓
外部数据源
```

### 5.2 前端技术栈

推荐：

- Next.js
- React
- TypeScript
- CSS / Tailwind CSS
- ECharts 或自定义轻量图表组件
- TanStack Query，真实 API 接入阶段使用

当前实现阶段优先目标：

- 初始化 Next.js 项目
- 使用 Mock 数据
- 不依赖真实后端
- 先跑通首页、成交日历、趋势报告、楼市利率页面

### 5.3 后端技术栈

最终后端建议：

- Go
- Gin / Chi
- MySQL
- Redis / Upstash Redis
- Cron / GitHub Actions / Vercel Cron

后端职责：

- 数据采集
- 数据清洗
- 数据存储
- 聚合查询
- BFF 接口保护
- 限流与防刷

### 5.4 数据库设计方向

核心表：

- city：城市表
- district：区域表
- house_transaction_daily：每日成交数据表
- interest_rate_daily：利率数据表
- data_source_sync_log：数据采集日志表

所有业务数据需要保留 city_id 字段，支持后续扩展到其他城市。

## 6. 安全设计

无登录注册场景下，无法完全阻止用户在浏览器开发者工具中看到接口请求。因此安全目标应调整为：

1. 不暴露原始敏感数据
2. 不允许批量拖取全量数据
3. 限制同一 IP 的访问频率
4. 不开放采集、管理、内部接口
5. 公开接口只返回页面所需聚合数据

推荐方案：

- 前端通过 BFF 请求后端
- Go 后端校验内部 API Key 或 HMAC 签名
- 使用 Origin / Referer allowlist
- 使用 Redis 进行 IP 限流
- 对趋势、日历等接口做缓存
