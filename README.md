# House Trend

上海房屋市场趋势分析看板。项目首期使用 Mock 数据打通网页访问、核心指标展示、趋势图表、成交日历、区域对比和楼市利率监控等产品闭环，后续可平滑接入 Go 后端、MySQL 历史数据与定时采集任务。

## 项目定位

- **分析范围**：首期聚焦上海市房屋市场，保留扩展到其他城市的能力。
- **核心指标**：成交均价、成交套数、成交面积、成交金额、区域热度、商业贷款利率、公积金贷款利率、首付比例和 SHIBOR 参考指标。
- **访问形态**：基于 Next.js 的响应式 Web 应用，适配 PC 与移动端。
- **当前阶段**：前端静态页面与 Mock 数据验证阶段。

## 功能模块

### 首页看板

- 城市、区域、房屋类型、时间范围筛选入口
- 成交均价、本月成交、成交面积、成交金额等核心指标卡片
- 成交均价趋势、成交量趋势图表
- 区域成交排行与市场核心摘要

### 成交数据

- 按自然月展示每日成交数据日历
- 展示每日成交套数、成交均价、成交面积、成交金额
- 提供月成交套数、面积、金额、均价汇总
- 通过热度底色识别成交活跃日期

### 趋势分析报告

- 展示城市成交金额与成交面积趋势
- 对比不同区域价格、成交与热度变化
- 输出规则型市场分析摘要

### 楼市利率

- 展示商业贷款、公积金贷款、首付比例等利率卡片
- 展示商业贷款利率、公积金贷款利率和 SHIBOR 趋势
- 提供利率变化对市场的简要解读

## 数据来源规划

项目当前使用 Mock 数据。后续计划接入以下公开数据来源并落库形成历史趋势分析能力：

1. 网上房地产：https://www.fangdi.com.cn/
2. 上海银行间同业拆放利率：https://www.shibor.sh.cn/shibor/
3. 上海公积金官网：https://www.shzfgjj.cn/

推荐的长期架构为前后端分离：Next.js 前端负责展示和 BFF 聚合，Go 后端负责数据采集、清洗、存储、聚合查询、安全控制与限流，MySQL 负责历史数据沉淀。

## 部署

项目面向 Vercel 部署，完成依赖安装后可直接使用 Next.js 默认构建流程：

```bash
npm run build
```

部署时建议配置后端 API 地址、访问限流策略和数据采集任务，当前 Mock 阶段无需额外环境变量。
